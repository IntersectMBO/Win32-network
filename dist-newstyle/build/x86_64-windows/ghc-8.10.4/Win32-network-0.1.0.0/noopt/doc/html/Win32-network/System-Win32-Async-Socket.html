connect ∷ Socket → SockAddr → IO () Source #

Unfortunatelly connect using interruptible ffi is not interruptible. Instead we run the connect in a dedicated thread and block on an MVar.

accept ∷ Socket → IO (Socket, SockAddr) Source #

This is a thin wrapper around accept. It's possible to killThread which runs the accept. It will leave a stranded thread, but closing a socket terminates accept call, and thus there's not resource leak.
