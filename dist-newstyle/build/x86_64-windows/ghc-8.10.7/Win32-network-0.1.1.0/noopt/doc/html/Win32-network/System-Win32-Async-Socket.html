connect ∷ Socket → SockAddr → IO () Source #

Unfortunately connect using interruptible ffi is not interruptible. Instead we run the connect in a dedicated thread and block on an MVar.

accept ∷ Socket → IO (Socket, SockAddr) Source #

This is a thin wrapper around accept, which runs it in a new thread to make it interruptible. When the thread that is blocked on this call is killed it will leave a stranded thread, but closing a socket terminates accept call, and thus there is no resource leak.
